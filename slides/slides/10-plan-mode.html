Plan Mode Is a Discipline

What Plan mode does

  • Read-only — Cursor cannot edit files
  • Proposes, asks questions, explores options
  • Forces you to know what you want before writing code
  • Surfaces assumptions before they become bugs

When to use it

  • Before implementing any non-trivial feature
  • When choosing an architecture or stack
  • When you're uncertain about the approach
  • When you want Cursor to ask before acting
The 5-minute plan prevents the 30-minute correction spiral.
