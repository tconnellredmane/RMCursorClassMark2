MCP Gives Cursor Hands

Without MCP

  • Cursor can only read files in your workspace
  • All knowledge comes from training data or what you paste in
  • No live data, no external tools

With MCP

  • Cursor can call APIs, query databases, run tools
  • Uses results live in the conversation
  • You control exactly what it can access

How it works

  1. MCP server exposes named tools
  2. Cursor discovers available tools
  3. Cursor decides when to call them
  4. Results flow back into the conversation

The mCase MCP exposes 5 tools: schema, all_datalists, datalist_by_name, datalist_by_id, all_relationships
